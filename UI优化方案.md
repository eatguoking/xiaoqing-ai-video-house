# 小晴的AI影视妙妙屋 UI 优化方案

本方案基于三个 UI 设计 skill：

- `layout-analyzer`：用于界面结构体检。
- `brand-guidelines-generator`：用于品牌和视觉规范统一。
- `infographic`：用于复杂信息区的流程化、看板化、时间线化设计。

## 1. 当前 UI 核心问题

### 1.1 layout-analyzer 结论

当前界面功能已经比较完整，但信息密度偏高，主要问题是：

1. 顶部工具栏按钮太多，项目、创作、调试、设置、导出混在一起。
2. 右侧检查器同时承担模型选择、输入、参数、输出预览、工作流操作，层级还不够清楚。
3. 底部 AI Jobs 和 Assets 在同一横向区域里，用户容易不知道“任务状态”和“生成结果”的关系。
4. 画布节点已有预览能力，但节点之间的生产链路还可以更直观。
5. API 配置页功能强，但对普通用户来说字段很多，需要更明显的配置步骤。
6. 中英文切换后，部分按钮文字可能变长，需要更稳定的工具栏分组。
7. 黑夜模式已经有基础，但部分浅色块和状态色还需要统一打磨。

## 2. 品牌视觉方向

### 2.1 brand-guidelines-generator 结论

产品定位：

```text
本地 AI 影视制作工作台
```

品牌气质：

```text
清晰、专业、温柔、轻巧、有创作感
```

视觉方向：

1. 不做营销网站风格。
2. 不做大面积渐变和装饰图形。
3. 优先做成桌面生产力工具。
4. 信息密集但要分区清楚。
5. 颜色服务于状态和工作流，不做花哨装饰。

### 2.2 推荐颜色系统

浅色模式：

```text
背景：#EEF1F5
面板：#FFFFFF
次级面板：#F7F8FB
主文字：#1D2430
弱文字：#667085
品牌强调：#0F8F8C
连接/同步：#3C6DF0
成功：#22885F
警告：#DE7D25
错误：#CE3D4A
角色/分镜辅助：#7B61D1
```

黑夜模式：

```text
背景：#10141D
面板：#171D28
次级面板：#202838
边框：#313B4F
主文字：#EEF3FB
弱文字：#9AA7BA
品牌强调：#22B8B4
连接/同步：#7AA2FF
成功：#4FC48D
警告：#F2A04B
错误：#FF6B7B
```

### 2.3 组件规范

1. 卡片圆角控制在 8px 左右。
2. 工具按钮使用图标 + 短文字。
3. 重要操作只保留一个主按钮色。
4. 节点类型用左侧色条区分，不要每个节点大面积上色。
5. 弹窗内部避免“卡片套卡片”，改成清楚的左右栏或表单分区。
6. 中英文切换按钮和夜间模式按钮固定在顶部右侧。

## 3. 信息架构优化

### 3.1 infographic 结论

这个软件的核心不是单页表单，而是一条 AI 视频生产链路：

```text
剧本 → 分镜 → 图片 → 视频 → 时间线 → 导出
```

界面应该让用户始终知道：

1. 我现在在哪一步。
2. 当前节点输入在哪里。
3. AI 输出在哪里。
4. 下一步应该点哪里。
5. 哪个 API 或模型正在工作。

## 4. 具体改版方案

## 4.1 顶部工具栏重构

当前：

```text
Projects / Storyboard / Library / Templates / Pipeline / Timeline / Debug / Save / API Config / Export
```

建议改成 5 个分组：

```text
项目区：Projects / Save
创作区：Storyboard / Library / Templates
流程区：Pipeline / Timeline
系统区：Debug / API Config
导出区：Export
```

右侧固定：

```text
中/EN
夜间/日间
```

优化目标：

1. 顶部不再像一排杂乱按钮。
2. 用户能快速找到“项目、创作、设置、导出”。
3. 中英文切换后不挤压标题。

## 4.2 右侧检查器重构

建议改成 4 个固定区块：

### A. 当前节点

显示：

```text
节点名称
节点类型
状态
复制 / 删除
```

### B. 选择模型

显示：

```text
模型搜索
模型来源分组
同步按钮
当前选中模型
```

无模型时：

```text
当前类型暂无可用模型
按钮：去配置 API
```

### C. AI 输入

显示：

```text
上游输出提示
使用上游输出
AI Input 文本框
生成参数：画幅、数量、时长、镜头
Run AI 主按钮
```

### D. 输出预览

显示：

```text
文本输出预览
图片/视频预览
打开大图/编辑文本
同步到资产库
```

优化目标：

1. 用户一眼知道在哪里输入。
2. 用户一眼知道哪里是输出。
3. Run AI 变成右侧最明确的主操作。

## 4.3 底部资产区重构

建议分成左右两块：

左侧：

```text
AI Jobs
任务状态：queued / running / succeeded / failed
模型名
耗时或更新时间
错误入口
```

右侧：

```text
AI Output / Assets
搜索
类型筛选
上传
资产卡片
```

资产卡片建议：

```text
缩略图 / 类型标签
标题
来源节点
生成时间
拖拽提示
```

优化目标：

1. 任务状态和最终资产分清。
2. 资产库更像可用素材池。
3. 支持拖拽到画布的行为更明显。

## 4.4 节点卡片优化

节点建议统一成：

```text
顶部：类型图标 + 标题 + 状态点
中部：输出预览
底部：模型名 + 简短摘要
左右：大尺寸连接点
```

节点类型色：

```text
剧本：青绿色
分镜：紫色
图片：绿色
视频：橙色
角色：粉紫色
声音：蓝紫色
导出：深灰
```

优化目标：

1. 画布上快速识别节点类型。
2. 生成结果能在节点里预览。
3. 连接点继续保持大点击范围。

## 4.5 API 配置页优化

当前 API 配置功能完整，但字段较多。

建议改成三段式：

### A. 左侧 API 分类

```text
剧本生成
图像生成
视频生成
```

每类显示：

```text
已配置数量
启用数量
是否有模型同步成功
```

### B. 中间 API 列表

每个 API 卡片显示：

```text
名称
Provider
Base URL
模型数量
启用状态
最近测试状态
```

### C. 右侧配置详情

建议分成折叠区：

```text
基础信息
接口路径
请求模板
响应路径
调试测试
```

优化目标：

1. 普通用户先看基础字段。
2. 高级字段可折叠。
3. API Debugger 更像“测试面板”。

## 4.6 Pipeline 流程面板优化

改成更清晰的流程卡：

```text
1 生成当前节点
2 创建下一个节点
3 拆分分镜
4 批量生成图片
5 保存项目
```

每一步显示：

```text
状态
可执行 / 不可执行原因
按钮
```

优化目标：

让新用户不需要理解所有节点，也能顺着流程完成一次视频制作。

## 4.7 Timeline 时间线优化

当前时间线偏列表。

建议改成横向片段轨道：

```text
视频片段卡片 → 视频片段卡片 → 视频片段卡片
```

每个片段显示：

```text
顺序号
缩略图
标题
时长
来源节点
```

后续可扩展：

```text
拖拽排序
设置时长
添加音频轨
导出剪辑工程
```

## 4.8 黑夜模式优化

当前已经有黑夜模式基础。

下一步重点：

1. 检查所有白色硬编码背景。
2. 提高输入框、资产卡片、弹窗边框对比度。
3. 调整状态色，避免夜间过亮。
4. React Flow 小地图和控制按钮统一暗色。
5. 图片/视频预览区域保持深色承托。

## 4.9 中英文模式优化

下一步重点：

1. 顶部按钮英文较长时自动压缩为图标 + tooltip。
2. 中文模式优先使用短词：项目、分镜、素材、流程、导出。
3. 英文模式保留专业术语：Storyboard、Assets、Pipeline。
4. 弹窗标题和表单字段全部进入统一字典。
5. 保证中文和英文都不撑破按钮。

## 5. 实施顺序

建议分 4 轮做：

### 第一轮：结构清爽

目标：

```text
顶部工具栏分组 + 右侧检查器重排 + 底部资产区分区
```

收益最大，用户马上能感到“不乱了”。

### 第二轮：节点和画布

目标：

```text
节点卡片升级 + 连接线视觉 + 输出预览更清楚
```

让无限画布更像真正的制作流程。

### 第三轮：API 配置页

目标：

```text
三分类更清楚 + 字段折叠 + 调试结果更可读
```

减少配置 API 时的压力。

### 第四轮：高级工作区

目标：

```text
Pipeline、Timeline、Export、Debug 全部重排
```

让制作流程从“能用”变成“顺手”。

## 6. 第一轮改版建议

第一轮我建议直接做这 6 项：

1. 顶部工具栏分组。
2. 右侧检查器改成“节点 / 模型 / 输入 / 输出”四段。
3. Run AI 按钮固定在 AI 输入区底部。
4. 底部 Jobs 和 Assets 视觉分栏。
5. 资产卡片增加“点击预览 / 拖拽到画布”的明显提示。
6. 暗色模式补齐第一批残留浅色块。

## 7. 验收标准

第一轮完成后，用以下标准检查：

1. 用户能在 5 秒内找到 AI 输入框。
2. 用户能在 5 秒内找到 AI 输出结果。
3. 用户能明确知道当前节点选了哪个模型。
4. 用户能看懂下一步该创建什么节点。
5. 中英文切换后顶部不乱。
6. 黑夜模式下没有刺眼白块。
7. 资产能继续点击预览、拖拽到画布。
8. 构建通过，桌面版能启动。
