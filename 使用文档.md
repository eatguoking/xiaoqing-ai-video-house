# 小晴的AI影视妙妙屋 使用文档

## 1. 软件简介

小晴的AI影视妙妙屋是一个本地运行的 AI 视频制作工作台。它用无限画布把剧本、分镜、图片、视频、角色、资产和导出流程串起来，适合做短剧、广告片、口播视频、分镜预演和 AI 视频素材管理。

这个项目不内置固定模型。剧本生成、图像生成、视频生成都通过外部 API 接入，用户可以自己配置不同平台、不同模型，并在画布节点里自由选择。

## 3. 主界面说明

软件主要分为四个区域：

1. 左侧节点库：添加 Script、Storyboard、Image、Video、Character、Voice、Export 等节点。
2. 中间无限画布：拖动、连接、组织 AI 视频制作流程。
3. 右侧检查器：选择模型、输入提示词、编辑节点内容、点击 Run AI 生成。
4. 底部资产区：显示 AI 输出结果、上传文件、查看任务状态、拖拽资产到画布。

## 4. 基本工作流

推荐的基础流程是：

1. 打开 API Config，先配置剧本、图像、视频 API。
2. 在画布中添加 Script 节点。
3. 选中 Script 节点，在右侧选择剧本模型。
4. 在 AI Input 输入剧情需求。
5. 点击 Run AI 生成剧本。
6. 点击 Create next node，把剧本传给 Storyboard 节点。
7. 在 Storyboard 节点生成分镜。
8. 点击 Split to images，把分镜拆成多个 Image 节点。
9. 选择图像模型，批量生成分镜图。
10. 选中图片节点，创建 Video 节点，调用视频 API 生成视频片段。
11. 在 Timeline 或 Export 中导出制作包。

## 5. API 配置

点击顶部的 API Config 打开配置中心。

API 分为三类：

1. Script API：用于剧本、分镜、角色设定等文本生成。
2. Image API：用于角色图、场景图、关键帧、分镜图生成。
3. Video API：用于文生视频、图生视频、短片段生成。

每一类下面都可以添加多个外部 API 服务。模型列表会从已配置 API 的 Models path 实时同步，不是写死在程序里的。

### 5.1 新增 API

在 API Config 中：

1. 选择 Script API、Image API 或 Video API。
2. 点击 Add。
3. 填写 Display name。
4. 填写 Provider，例如 openai、deepseek、runway、kling、replicate 等。
5. 填写 API Key。
6. 填写 Base URL。
7. 填写 Models path，通常是 `/models`。
8. 填写 Generate path。
9. 根据接口返回结构填写 Response path。
10. 点击 Save config。

### 5.2 常用字段解释

Display name：在模型列表里显示的 API 名称。

Provider：供应商名称，只用于标记和分组。

Fallback model：当接口无法返回模型列表时使用的备用模型名。

Base URL：接口基础地址，例如：

```text
https://api.openai.com/v1
```

Models path：获取模型列表的路径，常见为：

```text
/models
```

Generate path：生成接口路径，例如：

```text
/chat/completions
/images/generations
/videos/generations
```

Status path：异步视频任务查询路径。需要用 `{id}` 代表任务 ID，例如：

```text
/videos/generations/{id}
```

Request body template：请求体模板，可以使用变量：

```text
{{model}}
{{prompt}}
{{imageUrl}}
{{duration}}
{{ratio}}
{{size}}
{{count}}
```

Response content path：文本结果路径，例如：

```text
choices[0].message.content
```

Response URL path：图片或视频地址路径，例如：

```text
data[0].url
output.url
result.video_url
```

Task ID path：异步视频任务 ID 路径，例如：

```text
id
data.task_id
result.jobId
```

Status response path：异步任务状态路径，例如：

```text
status
data.state
```

Error response path：错误信息路径，例如：

```text
error.message
message
```

### 5.3 OpenAI 兼容文本接口示例

Script API 可以使用这个请求模板：

```json
{
  "model": "{{model}}",
  "messages": [
    { "role": "system", "content": "{{roleHint}}" },
    { "role": "user", "content": "{{prompt}}" }
  ],
  "temperature": 0.8
}
```

常用配置：

```text
Base URL: https://api.openai.com/v1
Models path: /models
Generate path: /chat/completions
Response content path: choices[0].message.content
Error response path: error.message
```

### 5.4 OpenAI 兼容图片接口示例

Image API 可以使用这个请求模板：

```json
{
  "model": "{{model}}",
  "prompt": "{{prompt}}",
  "size": "{{size}}",
  "n": {{count}}
}
```

常用配置：

```text
Base URL: https://api.openai.com/v1
Models path: /models
Generate path: /images/generations
Response URL path: data[0].url
Error response path: error.message
```

### 5.5 视频接口配置

如果视频接口是同步返回视频 URL，可以使用：

```json
{
  "model": "{{model}}",
  "prompt": "{{prompt}}",
  "image_url": "{{imageUrl}}",
  "duration": {{duration}},
  "ratio": "{{ratio}}"
}
```

同步视频接口常用配置：

```text
Generate path: /videos/generations
Response URL path: data[0].url
```

如果视频接口是异步任务，需要配置：

```text
Generate path: /videos/generations
Status path: /videos/generations/{id}
Task ID path: id
Status response path: status
Response URL path: output.url
```

状态值通常由外部 API 决定。常见完成状态包括 succeeded、completed、done、success。

### 5.6 API 调试

API Config 右下角有 API Debugger。

可以依次测试：

1. Test Base URL：检查 Base URL 和密钥是否能访问。
2. Sync models：检查模型列表能否读取。
3. Test generate：用测试提示词实际请求一次生成接口。

如果测试失败，优先检查：

1. API Key 是否正确。
2. Base URL 是否需要以 `/v1` 结尾。
3. Generate path 是否和平台文档一致。
4. 请求模板 JSON 是否合法。
5. Response path 是否能对应真实返回结构。

## 6. 节点使用说明

### 6.1 Script 节点

用于生成剧本、故事大纲、短剧台词、广告脚本等。

使用方式：

1. 左侧点击 Script。
2. 在右侧选择 Script API 里的模型。
3. 在 AI Input 输入需求。
4. 点击 Run AI。

生成结果会写入节点，并同步出现在底部 AI Output / Assets。

### 6.2 Storyboard 节点

用于把剧本拆成分镜。

常用方式：

1. 选中 Script 节点。
2. 点击 Create next node 创建 Storyboard。
3. 点击 Use as input 或直接生成。
4. 生成后可点击顶部 Storyboard 打开表格编辑器。

Storyboard 编辑器支持：

1. 编辑镜号、画面、台词、镜头运动、图片提示词、时长。
2. 导出 storyboard.csv。
3. 一键创建 Image 节点。

### 6.3 Image 节点

用于生成角色图、场景图、关键帧、分镜图。

使用方式：

1. 选中 Image 节点。
2. 选择 Image API 模型。
3. 输入画面描述。
4. 点击 Run AI。

生成图片会显示在节点预览和底部资产区。

### 6.4 Video 节点

用于生成视频片段。

使用方式：

1. 从 Image 节点创建 Video 节点，或手动添加 Video 节点。
2. 选择 Video API 模型。
3. 输入镜头运动、时长、画幅等要求。
4. 点击 Run AI。

如果外部视频 API 是异步任务，任务状态会显示在底部 AI Jobs。

### 6.5 Character 节点

用于保存角色设定、外貌、服装、性格、关系、视觉一致性说明。

Character 节点使用 Script API 生成文本内容，也可以在右侧手动编辑。

### 6.6 Voice 节点

当前 Voice 节点主要用于放置音频资产。语音生成 API 可以后续扩展为独立分类。

### 6.7 Export 节点

用于标记导出阶段。实际导出请使用顶部 Export 按钮。

## 7. 资产管理

底部 AI Output / Assets 区会显示所有 AI 生成结果和手动上传文件。

支持的资产类型：

1. Script：剧本文本。
2. Character：角色设定。
3. Scene：场景设定。
4. Image：图片。
5. Video：视频。
6. Audio：音频。

### 7.1 上传文件

点击底部 Upload，可以上传：

```text
图片、视频、音频、txt、md、json
```

也可以从桌面或文件夹直接把文件拖到画布上，程序会自动识别类型并创建对应节点。

### 7.2 预览和放大

点击资产卡片可以打开预览。

图片、视频、音频可以直接查看或播放。

文本资产可以在预览窗口里编辑，并保存修改。

### 7.3 拖拽到画布

从底部资产区拖拽资产到画布，会自动生成对应节点：

1. 图片资产生成 Image 节点。
2. 视频资产生成 Video 节点。
3. 音频资产生成 Voice 节点。
4. 文本资产生成 Script、Character 或普通文本类节点。

## 8. 项目管理

支持：

1. 新建项目。
2. 打开项目。
3. 重命名项目。

5. 保存当前画布。

点击顶部 Save 会保存当前项目的节点、连线和基础设置。

## 9. 提示词模板

点击顶部 Prompt Templates 可以保存常用提示词。

模板支持分类：

1. Script
2. Storyboard
3. Character
4. Image
5. Video
6. Voice

模板可以设置为：

1. Project：只在当前项目使用。
2. Global：全局通用。

点击 Apply 可以把模板内容填入右侧 AI Input。

## 10. 角色和场景库

点击顶部 Library 打开 Character / Scene Library。

可用于保存：

1. 固定角色设定。
2. 场景描述。
3. 视觉风格。
4. 人物服装和一致性信息。

在库中点击 Apply，会把内容应用到当前节点输入中。

## 11. Pipeline 流程面板

点击顶部 Pipeline 打开流程面板。

它提供常用流程按钮：

1. Generate current node：生成当前节点。
2. Create next node：创建下一个流程节点。
3. Split storyboard：把分镜拆成图片节点。
4. Generate image children：批量生成分镜图片。
5. Save project：保存项目。

## 12. Timeline 时间线

点击顶部 Timeline 打开时间线。

当前时间线会列出已经生成或上传的视频资产。

可以导出：

```text
Timeline CSV
```

## 13. 导出制作包

点击顶部 Export 打开导出窗口。

支持格式：

1. JSON：完整项目结构，适合二次处理。
2. Markdown：人类可读的制作包。
3. Storyboard CSV：分镜表。
4. Timeline CSV：视频资产时间线。
5. EDL：剪辑软件交换格式。
6. FCPXML：Final Cut Pro XML 格式。

建议交付给剪辑或制片时优先导出 Markdown、Storyboard CSV 和 Timeline CSV。

## 14. 调试日志

点击顶部 Logs 打开 API 调试日志。

日志会记录：

1. 调用的 API 类型。
2. Provider 和模型。
3. 请求地址。
4. HTTP 状态码。
5. 耗时。
6. 请求和响应摘要。
7. 错误信息。

如果 AI 生成失败，先看 Logs，再看 API Config 的 API Debugger。

## 15. 常见问题

### 15.1 点击 Run AI 没有生成

通常原因：

1. 当前节点没有选择模型。
2. 对应分类没有配置 API。
3. API 被关闭 Enabled。
4. API Key 错误。
5. Base URL 或 Generate path 不正确。
6. Request body template 不是合法 JSON。
7. Response path 没有匹配到返回内容。

处理方式：

1. 打开 API Config。
2. 点击 Sync models。
3. 点击 Test generate。
4. 打开 Logs 查看具体错误。

### 15.2 模型列表为空

可能原因：

1. 没有配置该节点类型对应的 API。
2. Models path 不正确。
3. 外部 API 不支持模型列表接口。
4. API Key 没有权限。

如果外部 API 没有模型列表接口，可以填写 Fallback model。程序会用这个模型名作为备用选项。

### 15.3 图片节点不显示图片

检查：

1. Image API 是否真的返回了图片 URL 或 b64_json。
2. Response URL path 是否正确。
3. 图片 URL 是否可以访问。
4. Logs 中是否有接口报错。

### 15.4 视频一直在任务中

异步视频 API 需要正确配置：

1. Task ID path。
2. Status path。
3. Status response path。
4. Response URL path。

如果平台返回的完成状态不是常见状态，需要根据该平台文档调整状态识别逻辑。

### 15.5 安装后打不开

查看启动日志：

```text
C:\Users\qin\AppData\Roaming\xiaoqing-ai-video-house\desktop-startup.log
```

常见原因：

1. 端口被占用，程序会自动尝试下一个端口。
2. 安装目录权限不足。
3. 数据库文件被占用。
4. 安全软件拦截本地服务。
